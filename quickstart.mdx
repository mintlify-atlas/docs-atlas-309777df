## Running the Application

After [installing](/installation) the prerequisites and building the project, you can start the expense tracking application using Maven:

```bash
mvn exec:java -Dexec.mainClass="com.mibanco.gastos.ui.Consola"
```

You will see the welcome screen:

```text
==================================================
   BIENVENIDO A LA GESTIÓN DE GASTOS DE MIBANCO
==================================================

--- MENÚ PRINCIPAL ---
1. Agregar gasto
2. Actualizar gasto
3. Eliminar gasto
4. Listar todos los gastos
5. Filtrar gastos por mes
6. Salir
Seleccione una opción:
```

## Your First Expense

Let's walk through adding your first expense to the system.

<Steps>
  <Step title="Select 'Add Expense' Option">
    At the main menu, type `1` and press Enter to add a new expense.
  </Step>

  <Step title="Enter Expense Title">
    Provide a descriptive title for your expense:
    
    ```text
    Título: Almuerzo en restaurante
    ```
    
    <Note>
      The title field is **required** and cannot be empty. The system will validate this before saving.
    </Note>
  </Step>

  <Step title="Enter Optional Reason">
    Add additional details about the expense (optional):
    
    ```text
    Motivo (Opcional): Reunión de equipo
    ```
    
    You can leave this blank by pressing Enter.
  </Step>

  <Step title="Enter Date">
    Provide the expense date in ISO format (YYYY-MM-DD):
    
    ```text
    Fecha (Formato AAAA-MM-DD): 2026-03-09
    ```
    
    <Warning>
      The date must be in **YYYY-MM-DD** format. If you enter an invalid format, the system will prompt you to try again with an example: `2026-03-02`
    </Warning>
  </Step>

  <Step title="Enter Amount">
    Enter the expense amount in Peruvian Soles:
    
    ```text
    Monto (S/): 45.50
    ```
    
    The amount will be automatically rounded to 2 decimal places using commercial rounding (HALF_UP).
  </Step>

  <Step title="Confirmation">
    After entering all data, you'll see:
    
    ```text
    Gasto agregado exitosamente
    ```
    
    Your expense has been saved to the `gastos.json` file automatically.
  </Step>
</Steps>

## Viewing Your Expenses

### List All Expenses

From the main menu, select option `4` to list all expenses:

```text
--- LISTA DE GASTOS ---
[a1b2c3d4] 2026-03-09 | Almuerzo en restaurante | Reunión de equipo | S/ 45.50
[e5f6g7h8] 2026-03-08 | Taxi al trabajo | Transporte diario | S/ 12.00
[i9j0k1l2] 2026-03-07 | Compras del supermercado | Despensa semanal | S/ 150.75
[m3n4o5p6] 2026-03-01 | Pago de luz |  | S/ 85.30
```

<Note>
  Expenses are automatically sorted by date in **descending order** (most recent first). Each expense displays an 8-character short ID that you can use for updates and deletions.
</Note>

### Filter by Month

To view expenses for a specific month, select option `5`:

```text
--- FILTRAR GASTOS POR MES ---
Ingrese el mes (1-12): 3
Ingrese el año (ejemplo: 2026): 2026
```

This will display all expenses from March 2026, sorted chronologically:

```text
[a1b2c3d4] 2026-03-09 | Almuerzo en restaurante | Reunión de equipo | S/ 45.50
[e5f6g7h8] 2026-03-08 | Taxi al trabajo | Transporte diario | S/ 12.00
[i9j0k1l2] 2026-03-07 | Compras del supermercado | Despensa semanal | S/ 150.75
[m3n4o5p6] 2026-03-01 | Pago de luz |  | S/ 85.30
```

## Updating an Expense

Found a mistake in an expense? You can update it using its 8-character ID.

<Steps>
  <Step title="Select Update Option">
    From the main menu, type `2` and press Enter.
  </Step>

  <Step title="Enter Expense ID">
    Copy the 8-character ID from your expense list:
    
    ```
    Ingrese el ID (8 caracteres) del gasto (o '0' para cancelar): a1b2c3d4
    ```
  </Step>

  <Step title="Enter Updated Information">
    Provide the new values for all fields:
    
    ```
    Nuevo Título: Almuerzo ejecutivo
    Nuevo Motivo (Opcional): Reunión con cliente importante
    Fecha (Formato AAAA-MM-DD): 2026-03-09
    Monto (S/): 65.00
    ```
  </Step>

  <Step title="Confirmation">
    ```
    Gasto actualizado exitosamente
    ```
  </Step>
</Steps>

## Deleting an Expense

To remove an expense from the system:

<Steps>
  <Step title="Select Delete Option">
    From the main menu, type `3` and press Enter.
  </Step>

  <Step title="Enter Expense ID">
    ```
    Ingrese el ID (8 caracteres) del gasto (o '0' para cancelar): e5f6g7h8
    ```
  </Step>

  <Step title="Confirm Deletion">
    The system requires confirmation before deleting:
    
    ```
    ¿Está seguro que desea eliminar este gasto? (S/N): S
    ```
    
    <Warning>
      Type `S` to confirm or `N` to cancel. Deletion is permanent and cannot be undone.
    </Warning>
  </Step>

  <Step title="Confirmation">
    ```
    Gasto eliminado exitosamente
    ```
  </Step>
</Steps>

## Understanding the Data Model

Each expense stored in `gastos.json` follows this structure:

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "titulo": "Almuerzo ejecutivo",
  "motivo": "Reunión con cliente importante",
  "fecha": "2026-03-09",
  "monto": 65.00
}
```

### Field Validation

The application enforces strict validation rules:

<CardGroup cols={2}>
  <Card title="Title Validation" icon="check">
    - Cannot be null or empty
    - Whitespace-only titles are rejected
  </Card>
  
  <Card title="Date Validation" icon="calendar">
    - Must be in ISO format (YYYY-MM-DD)
    - Cannot be null
    - Parsed using Java's LocalDate
  </Card>
  
  <Card title="Amount Validation" icon="money-bill">
    - Must be greater than zero
    - Automatically rounded to 2 decimals
    - Uses HALF_UP rounding mode
  </Card>
  
  <Card title="ID Handling" icon="hashtag">
    - Auto-generated UUID
    - 8-character short IDs for user operations
    - System prevents duplicate IDs
  </Card>
</CardGroup>

## Error Handling Examples

The application provides helpful error messages for common mistakes:

### Invalid Date Format

```
Fecha (Formato AAAA-MM-DD): 09-03-2026
Formato de fecha incorrecto, use AAAA-MM-DD (ejemplo: 2026-03-02)
```

### Invalid Amount

```
Monto (S/): abc
Monto inválido. Ingrese un número válido (ejemplo: 120.50)
```

### Empty Title

```
Título: 
Error de validación: El título del gasto no puede estar vacío
```

### Negative or Zero Amount

```
Monto (S/): -50
Error de validación: El monto del gasto debe ser mayor a cero
```

## Data Persistence

All expenses are automatically saved to `gastos.json` in the project root directory. The file is updated whenever you:

- Add a new expense
- Update an existing expense
- Delete an expense

<Note>
  The JSON file is human-readable and uses Jackson with JSR310 support for proper `LocalDate` serialization. You can back up this file to preserve your expense history.
</Note>
