## Introduction

The Gestión de Gastos Mibanco application implements a four-layer architecture that follows Clean Code principles and the Single Responsibility Principle (SRP). Each layer has a specific role and communicates through well-defined interfaces.

## Layer 1: Model Layer

The Model layer represents the core domain entities of the application. It contains pure data structures with minimal business logic.

### Gasto Entity

Located at `src/main/java/com/mibanco/gastos/model/Gasto.java`

```java
package com.mibanco.gastos.model;

import com.fasterxml.jackson.annotation.JsonFormat;
import java.math.BigDecimal;
import java.math.RoundingMode;
import java.time.LocalDate;
import java.util.UUID;

public class Gasto {
    private UUID id;
    private String titulo;
    private String motivo;
    
    @JsonFormat(shape = JsonFormat.Shape.STRING, pattern = "yyyy-MM-dd")
    private LocalDate fecha;
    
    private BigDecimal monto;

    public Gasto() {
        // Default constructor for Jackson deserialization
    }

    public Gasto(UUID id, String titulo, String motivo, 
                 LocalDate fecha, BigDecimal monto) {
        this.id = id;
        this.titulo = titulo;
        this.motivo = motivo;
        this.fecha = fecha;
        this.setMonto(monto); // Uses setter for automatic rounding
    }

    // Automatic rounding to 2 decimal places (HALF_UP)
    public void setMonto(BigDecimal monto) {
        if (monto != null) {
            this.monto = monto.setScale(2, RoundingMode.HALF_UP);
        }
    }

    @Override
    public String toString() {
        return String.format("[%s] %s | %s | %s | S/ %s",
                id.toString().substring(0, 8),
                fecha,
                titulo,
                motivo,
                monto);
    }
}
```

### Key Design Decisions

<Note>
The `Gasto` entity uses `BigDecimal` for financial precision instead of `double` or `float`, preventing floating-point arithmetic errors. The `setMonto()` method automatically rounds to 2 decimal places using `RoundingMode.HALF_UP` (commercial rounding).
</Note>

**Model Layer Responsibilities:**
- Define domain entities (`Gasto`)
- Encapsulate data with getters/setters
- Apply Jackson annotations for JSON serialization
- Enforce data consistency (e.g., automatic rounding)
- Format output representation (`toString()`)

## Layer 2: Repository Layer

The Repository layer abstracts data persistence, providing a clean interface for data access operations.

### Repository Interface

Located at `src/main/java/com/mibanco/gastos/repository/GastoRepository.java`

```java
package com.mibanco.gastos.repository;

import com.mibanco.gastos.model.Gasto;
import java.util.List;
import java.util.Optional;
import java.util.UUID;

public interface GastoRepository {
    List<Gasto> findAll();
    Optional<Gasto> findById(UUID id);

    void update(Gasto gasto);

}
```

### File-Based Implementation

Located at `src/main/java/com/mibanco/gastos/repository/FileGastoRepository.java`

```java
package com.mibanco.gastos.repository;

import com.fasterxml.jackson.core.type.TypeReference;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.datatype.jsr310.JavaTimeModule;
import com.mibanco.gastos.exception.GastoException;
import com.mibanco.gastos.model.Gasto;

import java.io.File;
import java.io.IOException;
import java.util.ArrayList;
import java.util.List;
import java.util.Optional;
import java.util.UUID;

public class FileGastoRepository implements GastoRepository {
    private static final String RUTA_ARCHIVO = "gastos.json";
    private final ObjectMapper objectMapper;

    public FileGastoRepository() {
        this.objectMapper = new ObjectMapper();
        // Register JavaTimeModule for LocalDate support
        this.objectMapper.registerModule(new JavaTimeModule());
    }

    private List<Gasto> loadFromFile() {
        File file = new File(RUTA_ARCHIVO);
        if (!file.exists() || file.length() == 0) {
            return new ArrayList<>();
        }
        try {
            return objectMapper.readValue(file, 
                new TypeReference<List<Gasto>>() {});
        } catch (IOException e) {
            throw new GastoException(
                "Error al leer el archivo de gastos: " + e.getMessage());
        }
    }

    private void saveToFile(List<Gasto> gastos) {
        try {
            objectMapper.writerWithDefaultPrettyPrinter()
                       .writeValue(new File(RUTA_ARCHIVO), gastos);
        } catch (IOException e) {
            throw new GastoException(
                "Error al guardar en el archivo de gastos: " + e.getMessage());
        }
    }

    @Override
    public List<Gasto> findAll() {
        return loadFromFile();
    }

    @Override
    public Optional<Gasto> findById(UUID id) {
        return findAll().stream()
                .filter(gasto -> gasto.getId().equals(id))
                .findFirst();
    }

    @Override
    public void save(Gasto gasto) {
        List<Gasto> gastos = loadFromFile();
        gastos.add(gasto);
        saveToFile(gastos);
    }

    @Override
    public void update(Gasto gastoActualizado) {
        List<Gasto> gastos = loadFromFile();
        for (int i = 0; i < gastos.size(); i++) {
            if (gastos.get(i).getId().equals(gastoActualizado.getId())) {
                gastos.set(i, gastoActualizado);
                saveToFile(gastos);
                return;
            }
        }
    }

    @Override
    public void deleteById(UUID id) {
        List<Gasto> gastos = loadFromFile();
        boolean eliminado = gastos.removeIf(gasto -> gasto.getId().equals(id));
        if (eliminado) {
            saveToFile(gastos);
        }
    }
}
```

- Define data access contracts (interface)
- Implement persistence mechanisms (file, database, etc.)

- Manage I/O exceptions
- Provide CRUD operations

## Layer 3: Service Layer

The Service layer contains the core business logic, validation rules, and orchestrates repository operations.

### GastoService Implementation

Located at `src/main/java/com/mibanco/gastos/service/GastoService.java`

```java
package com.mibanco.gastos.service;

import com.mibanco.gastos.exception.GastoException;
import com.mibanco.gastos.model.Gasto;
import com.mibanco.gastos.repository.GastoRepository;

import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.Comparator;
import java.util.List;
import java.util.UUID;

public class GastoService {
    private final GastoRepository repository;

    // Constructor-based dependency injection
    public GastoService(GastoRepository repository) {
        this.repository = repository;
    }

    public void agregarGasto(String titulo, String motivo, 
                            LocalDate fecha, BigDecimal monto) {
        validarDatosGasto(titulo, fecha, monto);
        
        Gasto nuevoGasto = new Gasto(UUID.randomUUID(), 
            titulo, motivo, fecha, monto);
        repository.save(nuevoGasto);
    }

    public void actualizarGasto(String idCorto, String titulo, 
                               String motivo, LocalDate fecha, BigDecimal monto) {
        UUID idReal = obtenerUUIDPorIdCorto(idCorto);
        
        Gasto gastoExistente = repository.findById(idReal)
                .orElseThrow(() -> new GastoException(
                    "No se encontró un gasto con el ID: " + idReal));
        
        validarDatosGasto(titulo, fecha, monto);
        
        gastoExistente.setTitulo(titulo);
        gastoExistente.setMotivo(motivo);
        gastoExistente.setFecha(fecha);
        gastoExistente.setMonto(monto);
        
        repository.update(gastoExistente);
    }

    public List<Gasto> listarGastos() {
        return repository.findAll().stream()
                .sorted(Comparator.comparing(Gasto::getFecha).reversed())
                .toList();
    }

    public List<Gasto> filtrarGastosPorMes(int mes, int anio) {
        if (mes < 1 || mes > 12) {
            throw new GastoException("El mes debe estar entre 1 y 12");
        }
        
        return repository.findAll().stream()
                .filter(gasto -> gasto.getFecha().getMonthValue() == mes 
                        && gasto.getFecha().getYear() == anio)
                .sorted(Comparator.comparing(Gasto::getFecha).reversed())
                .toList();
    }

    // Centralized business validation
    private void validarDatosGasto(String titulo, LocalDate fecha, 
                                   BigDecimal monto) {
        if (titulo == null || titulo.trim().isEmpty()) {
            throw new GastoException(
                "El título del gasto no puede estar vacío");
        }
        if (fecha == null) {
            throw new GastoException("La fecha del gasto es obligatoria");
        }
        if (monto == null || monto.compareTo(BigDecimal.ZERO) <= 0) {
            throw new GastoException(
                "El monto del gasto debe ser mayor a cero");
        }
    }

    // Helper method for user-friendly ID lookup
    private UUID obtenerUUIDPorIdCorto(String idCorto) {
        if (idCorto == null || idCorto.length() < 8) {
            throw new GastoException(
                "Debe ingresar al menos los primeros 8 caracteres del ID");
        }
        
        List<UUID> coincidencias = repository.findAll().stream()
                .map(Gasto::getId)
                .filter(id -> id.toString().startsWith(idCorto))
                .toList();
        
        if (coincidencias.isEmpty()) {
            throw new GastoException(
                "No se encontró ningún gasto que coincida con el ID proporcionado");
        }
        if (coincidencias.size() > 1) {
            throw new GastoException(
                "El ID ingresado coincide con múltiples gastos. " +
                "Ingrese más caracteres");
        }
        
        return coincidencias.get(0);
    }
}
```

### Business Logic Highlights

<Accordion title="Validation Rules (lines 73-83)">
  - Title must not be null or empty
  - Date is mandatory
  - Amount must be greater than zero
  
  All validation is centralized in `validarDatosGasto()` to ensure consistency across create and update operations.
</Accordion>

<Accordion title="Short UUID Lookup (lines 86-104)">
  Users can reference expenses by the first 8 characters of the UUID instead of the full 36-character string. This improves usability while maintaining uniqueness validation.
</Accordion>

<Accordion title="Stream API Usage (lines 56-69)">
  Java Streams are used for filtering and sorting expenses by date, demonstrating modern Java functional programming patterns.
</Accordion>

**Service Layer Responsibilities:**
- Enforce business rules and validation
- Generate UUIDs for new entities
- Orchestrate repository operations
- Transform data (e.g., short ID to UUID)
- Apply sorting and filtering logic
- Throw domain-specific exceptions

## Layer 4: UI Layer

The UI layer manages user interaction through a console-based interface.

### Console Implementation (Excerpt)

Located at `src/main/java/com/mibanco/gastos/ui/Consola.java`

```java
package com.mibanco.gastos.ui;

import com.mibanco.gastos.exception.GastoException;
import com.mibanco.gastos.model.Gasto;
import com.mibanco.gastos.repository.FileGastoRepository;
import com.mibanco.gastos.service.GastoService;

import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.Scanner;

public class Consola {
    private static final GastoService servicio = 
        new GastoService(new FileGastoRepository());
    private static final Scanner scanner = new Scanner(System.in);

    public static void main(String[] args) {
        boolean salir = false;
        
        System.out.println("==================================================");
        System.out.println("   BIENVENIDO A LA GESTIÓN DE GASTOS DE MIBANCO");
        System.out.println("==================================================");
        
        while (!salir) {
            mostrarMenu();
            try {
                String entrada = scanner.nextLine();
                int opcion = Integer.parseInt(entrada);
                
                switch (opcion) {
                    case 1 -> agregarGasto();
                    case 2 -> actualizarGasto();
                    case 3 -> eliminarGasto();
                    case 4 -> listarGastos();
                    case 5 -> filtrarGastos();
                    case 6 -> {
                        salir = true;
                        System.out.println("Gracias por usar el sistema!");
                    }
                    default -> System.out.println("Opción no válida");
                }
            } catch (NumberFormatException e) {
                System.out.println("Error. Ingrese un número válido");
            } catch (GastoException e) {
                // Handle business validation errors
                System.out.println("Error de validación: " + e.getMessage());
            } catch (Exception e) {
                System.out.println("Ocurrió un error inesperado: " + 
                    e.getMessage());
            }
        }
    }

    private static LocalDate leerFecha() {
        while (true) {
            System.out.print("Fecha (Formato AAAA-MM-DD): ");
            try {
                return LocalDate.parse(scanner.nextLine());
            } catch (DateTimeParseException e) {
                System.out.println(
                    "Formato de fecha incorrecto, use AAAA-MM-DD");
            }
        }
    }

    private static BigDecimal leerMonto() {
        while (true) {
            System.out.print("Monto (S/): ");
            try {
                return new BigDecimal(scanner.nextLine());
            } catch (NumberFormatException e) {
                System.out.println(
                    "Monto inválido. Ingrese un número válido");
            }
        }
    }
}
```

### UI Design Patterns

<Note>
The UI layer validates input **format** (e.g., valid date format, numeric input) while the Service layer validates **business rules** (e.g., amount > 0, title not empty). This separation ensures proper layering.
</Note>

**UI Layer Responsibilities:**
- Display menus and navigation
- Capture and validate user input format
- Handle parsing exceptions (dates, numbers)
- Catch and display business exceptions
- Format output for console display
- Manage application lifecycle (main loop)

## Cross-Layer Communication

The layers communicate through well-defined interfaces:

```
Consola (UI)
    ↓ calls method
GastoService (Service)
    ↓ calls interface method
GastoRepository (Repository Interface)
    ↓ implemented by
FileGastoRepository (Repository Implementation)
    ↓ uses
Gasto (Model)
```

### Example: Adding a New Expense

1. **UI Layer** (`Consola.java:66-79`): Captures user input
2. **Service Layer** (`GastoService.java:22-27`): Validates and creates entity
3. **Repository Layer** (`FileGastoRepository.java:62-66`): Persists to file
4. **Model Layer** (`Gasto.java:23-29`): Represents the data

This separation ensures that:
- Business logic is independent of UI implementation
- Data access is abstracted from business logic
- Each layer can be tested independently
- Changes in one layer don't cascade to others

## Summary

The layered architecture provides:

- **Clear separation of concerns**: Each layer has a single responsibility
- **Testability**: Dependency injection enables isolated unit testing
- **Maintainability**: Changes are localized to specific layers
- **Extensibility**: New implementations can be added without modifying existing code
